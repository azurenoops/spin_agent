# v1.26.0 — Boundary-Scoped Model (Feature 033)

**Released**: TBD

---

## Overview

Feature 033 introduces **authorization boundary definitions** as first-class entities, enabling systems to manage multiple named security perimeters with independent resource, component, and capability tracking.

---

## New Features

### Authorization Boundary Management
- **Boundary definitions**: Create, edit, and delete named security perimeters (Physical, Logical, Hybrid types)
- **Primary boundary protection**: Each system has exactly one primary boundary that cannot be deleted
- **Cascade handling**: Deleting a boundary reassigns resources, components, and mappings to the Primary boundary
- **Dashboard page**: Dedicated boundary management page with summary cards showing resource/component counts and coverage percentages

### Boundary-Scoped Compliance Tracking
- **Scoped capability mappings**: Capability-to-control mappings can target specific boundaries for per-boundary compliance tracking
- **Boundary-filtered gap analysis**: Gap analysis page includes boundary selector to filter results by specific boundary
- **Boundary comparison table**: Visual comparison of coverage percentages across boundaries with color-coded indicators

### Composite Narrative Generation
- **Multi-boundary narratives**: When a control has mappings across multiple boundaries, SSP narratives are auto-generated with organization-wide sections first, followed by per-boundary sections alphabetically
- **Manual override protection**: Manually customized narratives are preserved during propagation (logged as `CompositeNarrativeSkipped`)
- **Per-boundary tracking**: Narrative update response includes counts by boundary

### SSP §11 Authorization Boundary Section
- **Multi-boundary output**: SSP §11 now organizes content by boundary definition with name, type, description, resource table, and component inventory per boundary
- **Backward compatibility**: Single-boundary systems render identically to the previous flat format

### MCP Chat Tools
- `compliance_list_boundary_definitions` — List all boundaries for a system
- `compliance_create_boundary_definition` — Create with name, type, description
- `compliance_delete_boundary_definition` — Delete with reassignment to Primary
- `compliance_boundary_gap_analysis` — Boundary-scoped gap analysis with comparison
- `compliance_define_boundary` (modified) — Now supports `boundary_definition_name` parameter

### Azure Resource Discovery
- **Resource Graph integration**: Discover Azure resources via Resource Graph API
- **Suggested boundaries**: Resources grouped by resource group as suggested boundary definitions
- **Dedup detection**: Resources already in the boundary are flagged with badges
- **Dashboard UI**: "Discover Azure Resources" button with search filter, suggested boundary selection, and apply workflow

## Data Model Changes

### New Entity
- **`AuthorizationBoundaryDefinition`**: Named boundary container with `Name`, `BoundaryType` (Physical/Logical/Hybrid), `Description`, `IsPrimary`

### Modified Entities
- **`AuthorizationBoundary`**: Added nullable FK `AuthorizationBoundaryDefinitionId`
- **`SystemComponent`**: Added nullable FK `AuthorizationBoundaryDefinitionId`
- **`CapabilityControlMapping`**: Added nullable FK `AuthorizationBoundaryDefinitionId`

### Migration
- `20260316025720_Feature033_BoundaryScopedModel`: Creates boundary definitions table, adds FK columns, seeds primary boundary for existing systems, assigns existing resources/components

## API Changes

### New Dashboard Endpoints
| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/dashboard/systems/{systemId}/boundary-definitions` | List boundaries |
| POST | `/api/dashboard/systems/{systemId}/boundary-definitions` | Create boundary |
| PUT | `/api/dashboard/boundary-definitions/{id}` | Update boundary |
| DELETE | `/api/dashboard/boundary-definitions/{id}` | Delete boundary |
| GET | `/api/dashboard/systems/{systemId}/azure-discovery` | Discover Azure resources |
| POST | `/api/dashboard/systems/{systemId}/azure-discovery/apply` | Apply discovery results |

### Modified Endpoints
- **GET `/api/dashboard/systems/{systemId}/gaps`**: Accepts optional `boundaryDefinitionId` query parameter; returns `boundaryComparison` array when no filter
